<h3>Changements dans la version 1.42</h3>

<ul>

  <li>Ajout de détails dans la section «enregistrés» de la section «fonctionnement».</li>

  <li>Rajout des 25 dernière entrées dans la section « Enregistrés »</li>

  <li>rajout d’une limite de 300 mots exposés avec possibilité de déplier les notes plus longues en cliquant dessus « Enregistrés »</li>

</ul>

<h3>Changements dans la version 1.45</h3>

<ul>

  <li>Protection contre la perte de données : la sauvegarde «save» fusionne par date au lieu d’écraser, et les sauvegardes vides sont ignorées.</li>

  <li>Tolérance accrue : en cas de JSON corrompu, tentative de lecture depuis la sauvegarde .bak côté serveur.</li>

  <li>«Enregistrés» affiche maintenant toutes les notes (plus de limite).</li>

  <li>Autosauvegarde lors de la saisie et au changement d’émotion (avec tempo pour éviter les envois trop fréquents).</li>

  <li>PWA : stratégie de cache revue pour un affichage du calendrier instantané (cache d’abord sur le widget) tout en gardant les contenus dynamiques à jour.</li>

  <li>Montée de version Service Worker pour forcer l’actualisation sur mobile.</li>

</ul>

<h3>Changements dans la version 1.46</h3>

<ul>

  <li>Ajout d'un bouton «Modifier» dans la liste des notes enregistrées pour corriger ou compléter une ancienne note.</li>

</ul>

<h3>Changements dans la version 1.47</h3>

<ul>

  <li>Amélioration du mode édition : ajout des boutons «Enregistrer» et «Annuler» pour sécuriser les modifications.</li>

  <li>Correction de la navigation : le retour depuis la liste des notes enregistrées ramène toujours à la page de notes du jour.</li>

</ul>

<h3>Changements dans la version 1.48</h3>

<ul>

  <li>Correctif critique : correction d'un bug provoquant l'écrasement de la note du jour lors de l'édition d'unc ancienne note.</li>

  <li>Correctif d'affichage : suppression d'un titre en double dans la fenêtre d'édition.</li>

</ul>

<h3>Changements dans la version 1.49</h3>

<ul>

  <li>Correctif de stabilité : suppression d'une condition de course où l'autosauvegarde pouvait écraser la note du jour après une édition.</li>

  <li>Ajustement de mise en page : le titre «Notes enregistrées» s'affiche désormais correctement sur une seule ligne.</li>

</ul>
